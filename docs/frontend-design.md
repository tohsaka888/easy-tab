# Easy-Tab 前端设计文档（MVP）

> 参考：提供的截图风格（海绿纹理背景 + 卡片模块化桌面）。
> 对齐架构：响应式网格画布、模块化组件、编辑模式（拖拽/缩放/重排）。

## 1. 视觉定位与关键词

**定位**：轻盈的“桌面画布”体验，介于 Google 新标签页的效率感与 iOS 桌面编辑的熟悉度之间。整体为「玻璃卡片 + 纹理海绿背景 + 温润排版」。

**关键词**：海风、纸质卡片、轻玻璃、模块秩序、轻动画、易管理。

**记忆点**：大号时间 + 柔雾卡片 + 纹理背景 + 规整图标矩阵。

## 2. 信息层级与页面结构

1. **顶部时间层**：居中大号时间，日期行位于时间下方。
2. **搜索层**：时间下方的横向搜索条，具轻玻璃背景与阴影。
3. **模块画布层**：网格化卡片模块（天气/待办/便签/计时/快捷链接）。
4. **底部弱提示**：如同步状态或快捷提示。

## 3. 设计系统（Design Tokens）

### 3.1 色彩

- `--bg-1`: #0F5C58
- `--bg-2`: #0C4B49
- `--bg-3`: #0A3D3C
- `--surface-1`: rgba(255, 255, 255, 0.92)
- `--surface-2`: rgba(255, 255, 255, 0.85)
- `--surface-3`: rgba(255, 255, 255, 0.7)
- `--ink-1`: #0E1B1A
- `--ink-2`: #24413E
- `--accent-1`: #F8B547 (强调/按钮)
- `--accent-2`: #3BA6A6 (次强调)
- `--danger`: #E55353

### 3.2 字体

- **Display（时间/大数字）**：`"Big Shoulders Display"` 700
- **正文与中文**：`"LXGW WenKai"` 400/500
- **UI 小字**：`"IBM Plex Sans"` 400

> 若线上加载不便，可优先使用系统替代：`"PingFang SC"`, `"Hiragino Sans GB"`。

### 3.3 圆角与阴影

- 卡片圆角：12px（小卡）/ 16px（大卡）
- 搜索条圆角：999px
- 阴影：`0 8px 20px rgba(0,0,0,0.18)` + 内阴影 `inset 0 1px 0 rgba(255,255,255,0.6)`

### 3.4 纹理与背景

- 背景采用 **海绿渐变 + 低对比噪点纹理**。
- 轻微暗角（vignette）集中注意力到中心。

## 4. 网格与布局（对齐架构）

- 采用 **m × n** 等分网格，基于 `minCell` 动态计算。
- 默认安全内边距：`padding = 48px`（桌面），移动端为 `24px`。
- 推荐 **卡片对齐网格**：
  - 小卡：`w=2, h=2`
  - 中卡：`w=4, h=2`
  - 大卡：`w=4, h=3`
  - 超宽卡：`w=6, h=2`

**推荐默认排布（示例）**
- 第一行：天气（4x2）+ 事件卡（4x2）+ 便签（4x2）
- 第二行：快捷图标矩阵（8~12 个小卡）

## 5. 模块设计规范

### 5.1 搜索模块

- 位置：时间下方，横向居中。
- 样式：白色玻璃条 + 微弱边框 + 内部左对齐图标。
- 交互：聚焦时高亮边框，背景轻微变亮。

### 5.2 快捷访问模块

- 小方块卡片（2x2），图标居中。
- 悬停放大 1.03 倍，带柔光。

### 5.3 便签/待办模块

- 卡片边距统一 12px。
- 内部使用细线分隔与轻量列表样式。

### 5.4 时钟/倒计时

- 数字使用 Display 字体。
- 倒计时模块可加入小图标或表情增强亲和力。

## 6. 编辑模式（与架构一致）

- 进入编辑模式后：
  - 所有模块开启 **轻微晃动**（keyframes）。
  - 模块右上角显示删除按钮（红色圆点）。
  - 边框变为虚线高亮。
- 拖拽时出现 **半透明 ghost**。
- 缩放手柄：四角 10px 圆点，hover 时高亮。
- 不可放置区域以红色半透明遮罩提示。

## 7. 动效与交互

- 页面加载：时间/搜索/模块逐层淡入，采用 80~120ms stagger。
- Hover：卡片上浮 2px，阴影略增强。
- 拖拽：模块略微放大 + 透明度 0.8。

## 8. 响应式策略

- **桌面（>= 1200px）**：完整网格、密集排布。
- **平板（768~1199px）**：减少列数，保留核心模块。
- **手机（<= 767px）**：时间/搜索占主要视觉，模块纵向堆叠。

## 9. 可访问性

- 卡片文字对比度 >= 4.5:1。
- 支持键盘聚焦高亮。
- 交互热点区域 >= 40px。

## 10. 资产与实现建议

- 背景纹理使用 SVG/PNG 噪点叠加（5% 不透明度）。
- 卡片与按钮颜色全部来自 token，避免组件级硬编码。
- 动效优先 CSS keyframes 实现。
